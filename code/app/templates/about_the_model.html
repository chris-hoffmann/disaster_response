About the machine learning model

This project's motivation was to build a tool that supports the coordination of aid efforts during a natural disaster by identifying and categorizing help messages sent via social media channels. Our machine learning model classifies text messages into 34 distinctive (non-exclusive) categories, which allows for filtering out help messages and extracting the nature of a possible emergency. In an ideal scenario, the critical messages would then be forwarded to the relevant organization to accelerate aid delivery and prevent further damage.

The underlying data are publicly available from here. The merged and cleaned dataset employed for modeling is available as an SQLite database file, which can be downloaded from this GitHub repository. Feature extraction from text messages followed standard protocols: Word tokens were transformed to their base form using the WordNet Lemmatizer. Then, a count matrix of word tokens was generated. This matrix was subsequently converted into a tf-idf (term frequency times inverse document frequencies) representation that was employed for modeling. Among the several classifiers tested using Scikit-learn, random forests showed the best performance in terms of high recall. The parameters for feature extraction and classification were further refined by applying a random hyperparameter search. Further details regarding the selected hyperparameters and the performance of the model determined on a hold-out set can be found in the dedicated GitHub repository. Python scripts for data cleaning, hyperparameter tuning, and model evaluation are also available from there.

We embed our model into a web application (app) that was built using three excellent libraries; Flask for the backend, Bootstrap for the frontend, and Plotly for generating plots. The web app was then deployed on Heroku. My apologies for the less than perfect performance of the app. The free plan lacks in-memory cache, which does not make for great user experience.
